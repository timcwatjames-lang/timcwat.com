1. What is HTML?

HTML (HyperText Markup Language) is the standard language for creating web pages. It uses tags to structure content.

〈 Code
<h1>Hello, World!</h1>
<p>This is my first paragraph.</p>
▶ Output

Hello, World!

This is my first paragraph.

Every HTML Document Has the Same Basic Structure

This is called the HTML boilerplate. Every web page starts with this skeleton:

〈 HTML Boilerplate
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page</title>
</head>
<body>
    <h1>Welcome</h1>
    <p>Content goes here.</p>
</body>
</html>
💡 Tip: Every HTML tag comes in pairs: <tagname> opens and </tagname> closes. A few tags like <br> and <img> are self-closing.

HTML Tags Are Like Building Blocks

Think of HTML tags as containers that tell the browser what kind of content is inside. The browser then displays it accordingly.

〈 Tags Demo
<strong>Bold text</strong>
<em>Italic text</em>
<u>Underlined text</u>
<mark>Highlighted</mark>
<small>Smaller text</small>
▶ Output

Bold text

Italic text

Underlined text

Highlighted

Smaller text

2. Headings & Paragraphs

Headings create hierarchy. Paragraphs hold blocks of text. Together they form the backbone of your content.

Headings h1 to h6

HTML has six levels of headings. <h1> is the most important, <h6> the least. Use them to create a clear document outline.

〈 Headings
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
▶ Output

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5
Heading 6
⚠️ Important: Only use one <h1> per page (for the main title). Use <h2> through <h6> in order — don't skip levels. This helps screen readers and SEO.

Paragraphs & Text Formatting

〈 Paragraphs
<p>This is a paragraph. It wraps text in a block that starts on a new line with some space above and below.</p>
<p>This is another paragraph. Browsers automatically add margins between paragraphs.</p>
▶ Output

This is a paragraph. It wraps text in a block that starts on a new line with some space above and below.

This is another paragraph. Browsers automatically add margins between paragraphs.

〈 br & hr

<hr>
<p>Content after the horizontal rule.</p>

3. Links & Navigation

The <a> (anchor) tag creates hyperlinks to other pages, sections, or resources.

〈 Basic Link

▶ Output

Link Attributes

〈 Link Variants

<!-- Link to email -->
<a href="mailto:hello@example.com">Email Me</a>

<!-- Link to phone -->
<a href="tel:+1234567890">Call Us</a>

<!-- Link to a section on the page -->
<a href="#forms">Go to Forms</a>

4. Images & Media

Images make your page visual. The <img> tag is self-closing and requires at least the src and alt attributes.

〈 Image
<img src="https://picsum.photos/300/200"
     alt="A random placeholder image"
     width="300"
     height="200">
▶ Output

Image with Caption

Use the <figure> and <figcaption> tags to group an image with its caption.

〈 Figure
<figure>
    <img src="https://picsum.photos/300/200?random=1"
         alt="Sunset view">
    <figcaption>
        A beautiful sunset captured during golden hour.
    </figcaption>
</figure>
▶ Output
💡 Tip: Always include the alt attribute. It's used by screen readers for accessibility and shows when the image fails to load. Good alt text describes the image content.

Video

The <video> element embeds video files with controls for the user.

5. Lists

HTML provides three types of lists: unordered (bullets), ordered (numbers), and description lists.

Unordered List

Use <ul> for items where order doesn't matter. Each item is wrapped in <li>.

〈 Unordered List
<ul>
    <li>Apples</li>
    <li>Bananas</li>
    <li>Cherries</li>
    <li>Dates</li>
</ul>
▶ Output
  • Apples
  • Bananas
  • Cherries
  • Dates

Ordered List

Use <ol> when the sequence matters. Numbers are added automatically.

〈 Ordered List
<ol>
    <li>First, preheat the oven</li>
    <li>Mix the ingredients</li>
    <li>Bake for 30 minutes</li>
    <li>Let it cool before serving</li>
</ol>
▶ Output
  1. First, preheat the oven
  2. Mix the ingredients
  3. Bake for 30 minutes
  4. Let it cool before serving

Nested Lists

Lists can be placed inside other list items to create sub-lists.

〈 Nested List
<ul>
    <li>Fruits
        <ul>
            <li>Apple</li>
            <li>Banana</li>
        </ul>
    </li>
    <li>Vegetables
        <ol>
            <li>Carrot</li>
            <li>Broccoli</li>
        </ol>
    </li>
</ul>
▶ Output
  • Fruits
    • Apple
    • Banana
  • Vegetables
    1. Carrot
    2. Broccoli

Description List

Use <dl> for term-description pairs like glossaries or metadata.

〈 Description List
<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
</dl>
▶ Output
HTML
HyperText Markup Language
CSS
Cascading Style Sheets

6. Tables

Tables organize data into rows and columns. Use <table> with <tr> (rows), <th> (headers), and <td> (data cells).

〈 Simple Table
<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>Country</th>
    </tr>
    <tr>
        <td>Alice</td>
        <td>25</td>
        <td>USA</td>
    </tr>
    <tr>
        <td>Bob</td>
        <td>30</td>
        <td>UK</td>
    </tr>
</table>
▶ Output
Name Age Country
Alice 25 USA
Bob 30 UK

Table with Colspan & Rowspan

Cells can span multiple columns or rows using colspan and rowspan attributes.

〈 Colspan & Rowspan
<table>
    <tr>
        <th colspan="2">Full Width Header</th>
    </tr>
    <tr>
        <td rowspan="2">Spans 2 rows</td>
        <td>Row 1, Col 2</td>
    </tr>
    <tr>
        <td>Row 2, Col 2</td>
    </tr>
</table>
▶ Output
Full Width Header
Spans 2 rows Row 1, Col 2
Row 2, Col 2

8. Semantic HTML

Semantic tags give meaning to your content. They tell browsers and developers what each part of the page represents.

〈 Semantic Structure
<header>     <!-- Top of the page: logo, nav -->
<nav>         <!-- Navigation links -->
<main>        <!-- Primary content -->
    <section>   <!-- Themed content group -->
    <article>   <!-- Self-contained content -->
    <aside>     <!-- Sidebar / related content -->
<footer>      <!-- Bottom: copyright, links -->
header
nav
main
section
article
aside
footer
〈 Semantic vs Non-Semantic
<!-- Non-semantic (div soup) -->
<div class="header">...</div>
<div class="nav">...</div>

<!-- Semantic (meaningful) -->
<header>...</header>
<nav>...</nav>
💡 Why semantic HTML matters: It improves accessibility (screen readers use it), SEO (search engines understand your content), and maintainability (developers can read your code). Use <div> only when no semantic tag fits.

10. HTML Attributes

Attributes provide additional information about HTML elements. They go inside the opening tag and come in name-value pairs.

The id Attribute

The id attribute gives a unique identifier to an element. No two elements on a page should share the same id.

〈 id Attribute
<h1 id="page-title">Welcome</h1>
<p id="intro">This paragraph has a unique ID.</p>
▶ Output

Welcome

This paragraph has a unique ID.

The class Attribute

The class attribute assigns one or more class names to an element. Multiple elements can share the same class. Classes are primarily used for CSS styling and JavaScript selection.

〈 class Attribute
<p class="highlight important">
    This paragraph has two classes.
</p>
<p class="highlight">
    This paragraph shares the highlight class.
</p>
<p class="important">
    This paragraph shares the important class.
</p>
▶ Output

This paragraph has two classes.

This paragraph shares the highlight class.

This paragraph shares the important class.

The title Attribute

The title attribute provides extra information, usually shown as a tooltip on hover.

〈 title Attribute
<p title="This is a tooltip!">
    Hover your mouse over this text.
</p>
▶ Output (hover the text)

Hover your mouse over this text.

The data-* Attributes

Custom data-* attributes store extra information that JavaScript can access. They are invisible to users but very powerful for dynamic functionality.

〈 data-* Attributes
<button data-user-id="42"
        data-role="admin">
    User Profile
</button>

<div data-product="laptop"
     data-price="999">
    Laptop - $999
</div>
▶ Output
Laptop - $999
💡 Common global attributes: id, class, title, lang, dir (text direction), hidden (hides element), tabindex (keyboard order), spellcheck, style, data-*. Attributes always go in the opening tag.

11. HTML Entities & Special Characters

HTML entities let you display reserved characters (like < and >) that would otherwise be interpreted as HTML code. They always start with & and end with ;.

Why Entities Are Needed

If you type < in your HTML, the browser thinks you're starting a tag. To actually show the less-than symbol, you must write &lt;. The same applies to other reserved characters.

〈 Entities in Action
<p>
    Use &lt; for < and &gt; for >
</p>
<p>
    &amp; displays as &
    &quot; displays as "
</p>
<p>
    Non-breaking space: &nbsp;
    Copyright: &copy; 2026
</p>
▶ Output

Use < for < and > for >

& displays as & " displays as "

Non-breaking space:   Copyright: © 2026

Common HTML Entities Reference

〈 Entity Reference Table
<!-- Reserved Characters -->
&lt;    <   less-than
&gt;    >   greater-than
&amp;    &   ampersand
&quot;   "  double quote
&apos;   '   apostrophe (single quote)

<!-- Symbols & Punctuation -->
&copy;   ©  copyright
&reg;    ®  registered trademark
&trade;  ™ trademark
&cent;   ¢ cent
&pound;  £ pound
&euro;   € euro

<!-- Mathematical -->
&times;  × multiplication
&divide; ÷ division
&plusmn; ± plus/minus
&sect;   § section
&para;   ¶ paragraph sign

<!-- Spaces & Dashes -->
&nbsp;    non-breaking space
&ensp;    en space (half-width)
&emsp;    em space (full-width)
&mdash;   em dash
〈 Using Numeric Entities
<!-- Decimal entities -->
&#60;  <
&#62;  >
&#38;  &
&#169; ©

<!-- Hex entities -->
&#x3C; <
&#x3E; >
&#x2665; ♥ (heart)
&#x2605; star symbol
▶ Output

Decimal: < > & ©

Hex: < > ♥ ★

💡 Tip: Use &nbsp; (non-breaking space) when you want to prevent two words from breaking across lines, like "10 kg" or "Mrs. Smith". Use &lt; and &gt; whenever you display HTML code as text (like in this tutorial!).

12. Block vs Inline Elements

Every HTML element has a default display behavior. Block elements take up the full width available and start on a new line. Inline elements only take up as much width as needed and flow within text.

Block Elements

Block elements always start on a new line and stretch to fill the container width. They can contain other block elements and inline elements.

〈 Block Elements
<div style="background:lightblue">
    Div (block) takes full width
</div>
<p style="background:lightgreen">
    Paragraph (block) starts on new line
</p>
<h2 style="background:lightcoral">
    Heading (block) also on its own line
</h2>
▶ Output
Div (block) takes full width

Paragraph (block) starts on new line

Heading (block) also on its own line

Inline Elements

Inline elements sit beside each other on the same line. They only take up as much width as their content requires and cannot contain block elements.

〈 Inline Elements
<p>
    This is <span style="color:red">red text</span>
    and <a href="#">a link</a>
    and <strong>bold text</strong>
    all on the <em>same line</em>.
</p>
▶ Output

This is red text and a link and bold text all on the same line.

Inline-Block: The Best of Both Worlds

The display: inline-block value combines features of both. Elements sit beside each other (like inline) but you can set width, height, margin, and padding (like block).

〈 Inline-Block
<div style="display:inline-block;
     width:120px;background:salmon;
     padding:10px;text-align:center">
    Box 1
</div>
<div style="display:inline-block;
     width:120px;background:skyblue;
     padding:10px;text-align:center">
    Box 2
</div>
<div style="display:inline-block;
     width:120px;background:lightgreen;
     padding:10px;text-align:center">
    Box 3
</div>
▶ Output
Box 1
Box 2
Box 3
💡 Quick reference: Common block elements: <div>, <p>, <h1>-<h6>, <ul>, <ol>, <li>, <table>, <form>, <section>, <header>, <footer>. Common inline elements: <span>, <a>, <strong>, <em>, <img>, <code>, <br>.

13. Iframes & Embedded Content

The <iframe> element embeds another HTML page inside your current page. It's used for maps, videos, external widgets, and more.

Basic Iframe

An iframe creates a window into another page. Use the src attribute to specify the URL to embed.

〈 Iframe
<iframe
    src="https://example.com"
    width="100%"
    height="200"
    title="Example page">
    Your browser doesn't support iframes.
</iframe>
▶ Output

Iframe Attributes

Use allow for permissions, sandbox for security, loading="lazy" for performance, and srcdoc to embed HTML directly.

〈 srcdoc Attribute
<iframe
    srcdoc="<p>Hello from <b>srcdoc</b>!</p>"
    width="100%"
    height="60"
    title="srcdoc example">
</iframe>
▶ Output

Security with Sandbox

The sandbox attribute restricts what the embedded content can do. Use it when embedding untrusted content.

〈 Sandboxed Iframe
<!-- Restricted sandbox -->
<iframe
    src="https://example.com"
    sandbox
    title="sandboxed">
</iframe>

<!-- Permissive sandbox -->
<iframe
    src="https://example.com"
    sandbox="allow-scripts
             allow-same-origin"
    title="permissive">
</iframe>
▶ Output

Restricted: (content blocked by sandbox)

With permissions: (content allowed)

⚠️ Security note: Always use sandbox when embedding content you don't fully trust. Add permission values like allow-scripts, allow-same-origin, allow-forms only when needed. The loading="lazy" attribute defers loading until the iframe is near the viewport, improving page speed.

🌐 Interactive: Embedded Map (Concept)

Iframes are commonly used to embed Google Maps. Here's what the code looks like:

<iframe
    src="https://www.google.com/maps/embed?pb=..."
    width="100%"
    height="300"
    style="border:0;"
    allowfullscreen=""
    loading="lazy"
    title="Google Maps">
</iframe>

14. Meta Tags & SEO

Meta tags live inside the <head> section and provide information about your page to browsers, search engines, and social media platforms.

Essential Meta Tags

Every HTML page should include these basic meta tags for proper rendering and SEO.

〈 Basic Meta Tags
<!-- Character encoding -->
<meta charset="UTF-8">

<!-- Responsive viewport -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0">

<!-- Page title (shown in browser tab) -->
<title>My Awesome Page</title>

<!-- Description (shown in search results) -->
<meta name="description"
      content="Learn HTML with interactive examples.">

<!-- Keywords (optional — no longer used by most search engines) -->
<meta name="keywords"
      content="HTML, tutorial, web development, coding">

<!-- Author -->
<meta name="author"
      content="Your Name">

Open Graph Tags (Social Media)

OG tags control how your page appears when shared on Facebook, Twitter, LinkedIn, and other platforms.

〈 Open Graph Tags
<!-- Open Graph (Facebook, LinkedIn, etc.) -->
<meta property="og:title"
      content="My Awesome Page">
<meta property="og:description"
      content="Learn HTML interactively.">
<meta property="og:image"
      content="https://example.com/thumb.jpg">
<meta property="og:url"
      content="https://example.com/page">
<meta property="og:type"
      content="website">

<!-- Twitter Card -->
<meta name="twitter:card"
      content="summary_large_image">
<meta name="twitter:title"
      content="My Awesome Page">

Favicon & Theme Color

Set a favicon (browser tab icon) and theme color (mobile browser UI color) to brand your page.

〈 Favicon & Theme
<!-- Favicon (shown in browser tab) -->
<link rel="icon"
      href="favicon.ico"
      type="image/x-icon">

<!-- Apple touch icon (iOS home screen) -->
<link rel="apple-touch-icon"
      href="icon-192.png">

<!-- Theme color (mobile browser UI) -->
<meta name="theme-color"
      content="#1a1a2e">
💡 SEO tips: Keep your <title> under 60 characters and your <meta description> under 160 characters for optimal search result display. Use exactly one <h1> per page. Use descriptive URLs. Add alt text to all images.

15. HTML5 Features Overview

HTML5 is the latest major version of HTML. It introduced new semantic elements, multimedia support, graphics capabilities, and powerful browser APIs.

What HTML5 Brought

Before HTML5, developers used <div> for everything and relied on plugins like Flash for video and audio. HTML5 made the web platform richer and more standardized.

〈 HTML5 Semantic Elements
<!-- NEW in HTML5 -->
<header>    Page header
<nav>       Navigation
<main>      Primary content
<section>   Themed group
<article>   Self-contained piece
<aside>     Sidebar content
<footer>    Page footer

<!-- Forms -->
<input type="email">
<input type="date">
<input type="range">
<input type="color">
<datalist>  Autocomplete options
<output>    Calculation result

HTML5 introduced <video> and <audio> elements, replacing the need for Flash or other plugins.

Browser APIs (Structures for JS)

HTML5 also defined APIs that JavaScript can use. Here are some of the most powerful ones:

Geolocation API

Get the user's current location. Used for maps, nearby places, weather apps, and location-based services.

Local Storage API

Store data in the user's browser persistently. Unlike cookies, data is never sent to the server.

Drag & Drop API

Allow users to drag elements around the page. Used in file uploaders, kanban boards, and sortable lists.

History API

Manipulate the browser's history stack. Powers single-page applications (SPAs) without full page reloads.

💡 HTML5 Browser Support: All modern browsers fully support HTML5. You can safely use these elements and APIs today. For older browsers (IE11 and below), you may need polyfills or fallbacks for some features like <canvas> and the Geolocation API.
